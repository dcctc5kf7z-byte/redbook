# V3.3 设计稿：拆解模式 + 挖掘模式

> **版本**：V3.3 Draft
> **日期**：2026-07-01
> **目标**：补全产品从"帮你写"到"教你写"的关键能力缺口
> **状态**：已修复（全员自审后 9 项修复）

---

## 一、背景与目标

### 当前问题

V3.2.0 的 /redbook 是一个"内容加工工具"——用户带素材来，系统帮ta打磨成成品。但用户最痛苦的两个问题，系统没有回答：

1. **"这篇爆款为什么爆？"** → 缺少拆解能力
2. **"我不知道写什么"** → 缺少挖掘能力

### 设计目标

| 目标 | 衡量标准 |
|------|---------|
| 用户能从一篇爆款中提取可复用的结构 | 拆解输出 ≥5 个可复用要素 |
| 用户能从模糊想法中找到具体选题 | 挖掘输出 ≥3 个可写角度 |
| 两个模式与现有系统无缝集成 | 不破坏现有 Fact Sheet + 人设路由 + 合规模块 |
| 触发词自然，用户无需学习 | 触发词 ≤3 个字，语义明确 |

## 二、拆解模式（Content Analysis Mode）

### 2.1 触发词

| 触发词 | 场景 |
|--------|------|
| `拆解这篇笔记` | 用户直接贴一篇笔记 |
| `帮我分析一下这篇` | 用户贴笔记 + 简短说明 |
| `这篇为什么能爆` | 用户贴笔记 + 明确意图 |
| `为什么这篇数据好` | 用户贴笔记 + 关注数据 |

**触发逻辑**（双重条件，必须同时命中）：

| 条件 | 判定规则 | 示例 |
|------|---------|------|
| 条件1：关键词命中 | 消息中包含"拆解"/"分析"/"为什么爆"/"为什么火"/"为什么数据好" | "帮我拆解这篇" ✅ / "分析一下" ✅ / "帮我写" ❌ |
| 条件2：笔记内容特征 | 满足以下任意 2 项：①有标题格式（"标题：xxx"或首行短于20字）②有标签（含 #）③正文超过 100 字④有"正文：""标签："等结构标记 | "标题：xxx\n正文：xxx\n标签：#xx" ✅（3/4）/ "这个产品好用吗" ❌（0/4） |

**误触发防护**：
- 仅命中条件1、不命中条件2 → 不进入拆解模式，走写作模式（如"帮我分析一下这个产品"→ 实际是想写推荐笔记）
- 仅命中条件2、不命中条件1 → 不进入拆解模式，走写作模式（如用户贴了一段笔记说"帮我优化"）
- 条件1 + 条件2 同时命中 → 进入拆解模式

**模式冲突处理**：
- "帮我拆解这篇笔记然后写一篇类似的" → 先执行拆解，拆解完成后自动询问"要基于哪个可复用要素写一篇？"
- "帮我拆解这篇笔记的标题" → 命中"拆解"但条件2不满足（用户只要标题）→ 走写作模式的标题分析，不进入拆解模式

### 2.2 输入格式

用户直接粘贴一篇小红书笔记的完整内容：

```
标题：xxx
正文：xxx
标签：#xxx #xxx
```

或直接粘贴截图内容（如果用户使用 OCR）。

系统不需要用户填写 Fact Sheet——因为拆解的对象是别人的笔记，不是帮用户写笔记。

### 2.3 输出结构

**快速拆解（默认）**——3 个核心维度，手机友好：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📝 拆解报告：[笔记标题]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

一、钩子拆解
├─ 钩子类型：[悬念型/数字型/痛点型/身份型/场景型/对比型/疑问型]
├─ 钩子强度：[1-5分] — [一句话为什么有效]
└─ 可复用公式：[提炼出的通用公式]

二、结构拆解
├─ 整体结构：[四段式/故事体/清单体/对比体/问答体]
├─ 段落功能：[哪段做什么，一句话概括]
└─ 结构亮点：[这个结构有什么特别之处]

三、收藏触发点
├─ 触发句："[最可能让人想收藏的那句话]"
└─ 触发机制：[实用价值 / 情感共鸣 / 信息差 / 稀缺性 / 身份认同]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
💡 回复"详细"查看完整拆解（情绪曲线 + 干货分析 + 互动分析 + 可复用清单）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**标准拆解（用户回复"详细"后）**——7 个维度，含量化数据：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📝 完整拆解：[笔记标题]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

四、情绪曲线
├─ 整体走势：[高开低走 / 低开高走 / 波浪型 / 平稳型 / 反转型]
├─ 情绪强度：[1-5分] — [情绪高点在第几句]
└─ 情绪节奏：[快节奏冲击 / 慢节奏沉浸 / 节奏变化]

五、干货分析
├─ 干货密度：[0.0-1.0] — [高/中/低]
├─ 干货类型：[经验型/数据型/步骤型/对比型/资源型]
└─ 干货分布：[前置集中 / 均匀分布 / 结尾爆发]

六、互动分析
├─ 评论区预判：[读者可能在评论区讨论什么]
├─ 互动钩子：[笔记中是否有引导评论的设计]
└─ 分享动力：[读者为什么会转发这篇笔记]

七、可复用清单
├─ 标题公式：[从这篇笔记提炼的标题公式]
├─ 结构模板：[可直接套用的结构]
├─ 金句模板：[可改写的金句]
├─ 钩子模板：[可复用的钩子]
└─ 避坑提醒：[这篇笔记有什么可以做得更好的地方]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 拆解量化数据
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
| 维度 | 数值 | 参考基准 |
|------|------|---------|
| 钩子强度 | X/5 | 同类型均值 X.X |
| 情绪强度 | X/5 | 同类型均值 X.X |

```

**深度拆解（用户说"深度拆解"时）**——在标准拆解基础上增加：同类型笔记对比 + 改写建议 + 优化方案。

### 2.3.1 量化标准（对齐 viral-patterns.jsonl）

| 维度 | 量化字段 | 取值范围 | 计算方式 |
|------|---------|---------|---------|
| 钩子强度 | hook_strength | 1-5 | 悬念/数字/痛点/身份/场景/对比/疑问 — 命中1个类型+1分，有情绪词+1分，≤20字+1分 |
| 情绪强度 | emotion_strength | 1-5 | 前3句有情绪钩子+1分，有"我"的故事+1分，有感官细节+1分，有转折+1分，有金句+1分 |
| 干货密度 | substance_density | 0.0-1.0 | 干货句数/总句数（判定规则同 SKILL.md 干货句测量协议） |
| 结构类型 | structure_type | 枚举 | 四段式/故事体/清单体/对比体/问答体 |
| 标题类型 | title_type | 枚举 | 数字清单/疑问钩子/对比冲突/情绪共鸣/悬念故事（对齐 viral-patterns.jsonl） |
| 开头钩子 | hook_type | 枚举 | 直接切入/悬念/共鸣/场景/反问（对齐 viral-patterns.jsonl） |
| 内容类型 | content_type | 枚举 | 好物推荐/教程攻略/情感共鸣/探店打卡/对比测评（对齐 SKILL.md） |

### 2.3.2 数据飞轮回写

**自动写入规则**：每次拆解完成后，将量化数据写入 `data/viral-patterns.jsonl`（与现有 schema 对齐）。

**写入格式**：
```json
{
  "date": "2026-07-01",
  "content_type": "好物推荐",
  "title_type": "数字清单",
  "hook_type": "悬念",
  "emotion_curve": "低开高走",
  "substance_density": 0.35,
  "sentence_style": "口语化",
  "greeting": "姐妹们",
  "emoji_density": 0.25,
  "hook_level": "L4",
  "personal_marker": "我上次",
  "source": "analysis_mode",
  "hook_strength": 4,
  "emotion_strength": 3,
  "structure_type": "四段式"
}
```

**铁律**：
- 拆解量化数据**自动写入** viral-patterns.jsonl（不等用户反馈）
- 文件不存在时**自动创建**
- 写入失败不阻断拆解输出（异步写入）
- `source` 字段标记为 `"analysis_mode"`，与手动生成区分

### 2.4 拆解深度分级

| 级别 | 触发条件 | 拆解深度 |
|------|---------|---------|
| 快速拆解 | 用户只贴了笔记，没有额外说明 | 只输出"钩子 + 结构 + 收藏触发点"三个核心维度 |
| 标准拆解 | 用户说"帮我拆解"或"分析一下" | 输出全部七个维度 |
| 深度拆解 | 用户说"深度拆解"或"详细分析" | 在标准拆解基础上，增加：同类型笔记对比、改写建议（经过合规过滤）、优化方案 |

### 2.5 与现有系统的集成

```
用户输入一篇笔记
    ↓
触发词检测 → 命中"拆解"类关键词
    ↓
进入拆解模式（跳过 Fact Sheet）
    ↓
输出拆解报告
    ↓
[可选] 用户说"帮我写一篇类似的" → 自动进入写作模式
    ↓
将拆解结果中的"可复用清单"作为 Fact Sheet 的预填数据
    ↓
走正常的 Fact Sheet → 人设路由 → 合规 → 输出流程
```

**关键设计决策**：
- 拆解模式**跳过 Fact Sheet**——因为拆解的是别人的笔记，不需要用户的事实信息
- 拆解结果可以**无缝衔接到写作模式**——用户说"帮我写一篇类似的"，拆解结果自动变成写作的输入
- 拆解模式**不需要人设路由**——因为拆解是分析，不是生成

## 三、挖掘模式（Content Mining Mode）

### 3.1 触发词

| 触发词 | 场景 |
|--------|------|
| `我不知道写什么` | 用户完全没有方向 |
| `帮我想几个选题` | 用户有模糊方向 |
| `这个话题怎么写` | 用户有话题，不知道怎么切入 |
| `有什么可以写的` | 用户想找灵感 |

**触发逻辑**（双重条件，必须同时命中）：

| 条件 | 判定规则 | 示例 |
|------|---------|------|
| 条件1：关键词命中 | 消息中包含"选题"/"写什么"/"不知道写什么"/"怎么写"/"有什么可以写"/"帮我想" | "帮我想几个选题" ✅ / "写什么好" ✅ / "帮我写" ❌ |
| 条件2：无具体素材 | 用户没有提供产品名/店名/主题等具体事实（Fact Sheet 必填项缺失 ≥2 个） | "想写关于职场的东西" ✅（无具体素材）/ "推荐珀莱雅双抗精华" ❌（有产品名） |

**误触发防护**：
- 仅命中条件1、有具体素材 → 不进入挖掘模式，走写作模式（如"这个话题怎么写：推荐珀莱雅"→ 有产品名，走写作）
- 仅命中条件2、无关键词 → 不进入挖掘模式，走写作模式的输入校验（如用户只说了"帮我写"→ 走现有流程）
- 条件1 + 条件2 同时命中 → 进入挖掘模式

### 3.2 输入格式

用户输入一个模糊的想法或话题：

```
"我最近在用一个很牛的面霜"
"想写关于职场的东西"
"小红书上减肥很火，我能写什么"
"我是做烘焙的，想发小红书"
```

系统需要**反向提问**来收集信息——这是挖掘模式与写作模式的核心区别。

### 3.3 输出结构（两阶段）

#### 阶段一：提问（系统主动问 — 默认只问 2 个核心问题）

系统根据用户输入，**默认只问 2 个最关键的问题**（降低流失率）：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔍 选题挖掘：[用户的话题]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

两个问题，帮你找到最能爆的角度：

1. 你在这个领域有什么独特的经历或视角？
   （比如：用了3年、踩过坑、有专业背景、有独特发现）

2. 你希望这篇笔记达到什么效果？
   （涨粉 / 带货 / 纯分享 / 建立人设 / 引发讨论）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
💡 回答这两个就能出方案。想更精准？补充：目标读者是谁、有什么素材可以用。
💡 不想回答？回复"直接给我选题"，我基于通用角度出方案。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**提问规则**：
- 默认问 2 个核心问题（独特经历 + 目标效果）
- 用户回答后 → 直接输出选题方案（不追问）
- 用户补充了更多信息 → 在选题方案中体现（更精准的角度）
- 用户说"直接给我选题" → 跳过提问，基于通用角度输出（标注"角度基于通用分析，补充你的经历后可更精准"）
- 深度挖掘（用户说"帮我做一个选题规划"）→ 问 4 个问题（增加：目标读者 + 可用素材）

#### 阶段二：输出选题方案

用户回答后，系统输出：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
💡 选题方案：[用户的话题]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

基于你的信息，我找到了 5 个可写角度：

━━━ 角度1：[角度名称] ━━━
├─ 选题：[具体选题]
├─ 标题：[推荐标题]
├─ 人设：[推荐人设 + 理由]
├─ 结构：[推荐结构 + 段落大纲]
├─ 预期爆款潜力：★★★★☆
├─ 爆款理由：[为什么这个角度可能爆]
├─ 风险提示：[这个角度有什么风险]
└─ 参考特征：[基于 viral-patterns.jsonl 同类型高频特征，如"标题用数字法+悬念法的组合，情绪曲线低开高走"]

━━━ 角度2：[角度名称] ━━━
...

━━━ 角度3：[角度名称] ━━━
...

━━━ 角度4：[角度名称] ━━━
...

━━━ 角度5：[角度名称] ━━━
...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 选题对比
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

| 角度 | 爆款潜力 | 难度 | 适合人设 | 推荐理由 |
|------|---------|------|---------|---------|
| 角度1 | ★★★★☆ | 低 | 闺蜜型 | 最容易上手 |
| 角度2 | ★★★★★ | 中 | 讲师型 | 爆款潜力最高 |
| 角度3 | ★★★☆☆ | 高 | 专家型 | 适合长期人设 |
| 角度4 | ★★★★☆ | 低 | 搞笑型 | 互动率最高 |
| 角度5 | ★★★★☆ | 中 | 种草型 | 带货转化最好 |

💡 我的建议：如果你是新手，从角度1开始；如果你想要爆款，选角度2。

⚠️ 爆款潜力基于历史数据参考（viral-patterns.jsonl），实际效果取决于内容质量和发布时机。

```

### 3.4 挖掘深度分级

| 级别 | 触发条件 | 挖掘深度 |
|------|---------|---------|
| 快速挖掘 | 用户说"帮我想几个选题" | 直接输出 3-5 个角度，不提问 |
| 标准挖掘 | 用户说"我不知道写什么" | 先问 3 个问题，再输出 5 个角度 |
| 深度挖掘 | 用户说"帮我做一个选题规划" | 问 5 个问题，输出 10 个角度 + 一周内容排期 |

### 3.5 与现有系统的集成

```
用户输入模糊想法
    ↓
触发词检测 → 命中"挖掘"类关键词
    ↓
进入挖掘模式（跳过 Fact Sheet）
    ↓
系统提问 → 用户回答
    ↓
输出选题方案
    ↓
[可选] 用户说"帮我写角度2" → 自动进入写作模式
    ↓
将角度2的标题、人设、结构作为 Fact Sheet 的预填数据
    ↓
走正常的 Fact Sheet → 人设路由 → 合规 → 输出流程
```

**关键设计决策**：
- 挖掘模式**先提问再输出**——这是与写作模式的核心区别
- 提问的目的是**找到用户的独特视角**——不是收集事实，而是发现角度
- 挖掘结果可以**无缝衔接到写作模式**——用户选一个角度，系统自动进入写作

## 八、风险与应对

| 风险 | 应对 |
|------|------|
| 拆解质量不稳定 | 每个维度有量化标准（2.3.1 节），拆解后自检：是否输出≥3个可复用要素 |
| 挖掘模式提问导致用户流失 | 默认只问2个核心问题 + 提供"直接给我选题"跳过选项 |
| 模式检测误触发 | 双重条件判定（关键词+内容特征），误触发防护规则（见2.1节） |
| Token 消耗增加 | 默认快速拆解（3维度），标准/深度需用户主动请求；懒加载 analysis.md/mining.md |
| 与现有功能冲突 | 严格模式隔离 + 20个测试用例覆盖（含模式冲突测试T18-T20） |
| 深度拆解的改写建议含红线词 | 改写建议经过合规过滤（标题/正文/标签三区域独立扫描），与写作模式一致 |
| 拆解非笔记内容 | 内容特征校验（条件2），不满足时提示"这不是小红书笔记格式" |
| 长文拆解输出过长 | 超过500字的笔记自动分段分析，输出时按段落折叠，不一次性全量输出 |

## 十一、总结

| 模式 | 核心价值 | 技术复杂度 | 优先级 |
|------|---------|-----------|--------|
| 拆解模式 | "这篇为什么爆" → 可复用的结构 + 量化数据 + 数据飞轮回写 | 低（不改架构） | 🔴 最高 |
| 挖掘模式 | "我不知道写什么" → 具体选题（2问精简 + 跳过选项） | 低（不改架构） | 🔴 最高 |

**一句话总结**：拆解模式让用户"从爆款中学习"，挖掘模式让用户"从零开始找到方向"。两个模式合在一起，产品从"帮你写"升级为"教你写"。
