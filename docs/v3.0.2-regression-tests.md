## TC-V3.0.2-01：软硬约束双轨制（AI 高频词黑名单）

**修复内容**：12 个 AI 高频词黑名单，输出前强制扫描替换，不依赖 LLM 自觉。

**测试输入**：
```
帮我写一条好物推荐，推荐珀莱雅双抗精华，用了3个月，皮肤真的变好了，价格不到200
```

**验证点**：
1. 输出正文中不包含以下 12 个黑名单词：
   - 值得注意的是、需要指出、总的来说、综上所述
   - 除此之外、与此同时、不难发现、由此可见
   - 在一定程度上、不可否认、具体而言、换句话说
2. 若 LLM 生成时意外使用了黑名单词，系统在输出前强制替换
3. 替换后上下文连贯性不受影响

**验收标准**：
- ✅ PASS：输出正文 0 个黑名单词
- ❌ FAIL：输出正文出现 ≥1 个黑名单词

## TC-V3.0.2-02：路由兜底 seeder + 决策链路日志

**修复内容**：所有信号均无法判定时兜底到 `seeder`；每次路由决策记录 `routing_trace`。

**测试输入 A（兜底场景）**：
```
帮我写点东西
```
- 无内容类型信号、无用户名信号、无 Fact Sheet 偏好

**验证点 A**：
1. 系统无法判定内容类型 → 提示用户补充信息（不直接兜底生成）
2. 若进入生成流程，人设兜底到 `seeder`
3. routing_trace 记录："无→无→无→无→兜底seeder（所有信号为空）"

**测试输入 B（正常路由 + 日志）**：
```
推荐一款防晒霜，用了半年了
```
- 用户名无特殊信号，内容类型=好物推荐

**验证点 B**：
1. 人设路由到 `seeder`（好物推荐默认）
2. routing_trace 格式正确："无→无→无→好物推荐默认→路由seeder"
3. routing_trace 写入反馈数据的 `routing_trace` 字段

**验收标准**：
- ✅ PASS：兜底场景路由到 seeder + routing_trace 格式正确
- ❌ FAIL：兜底场景路由到 NULL/随机人设，或 routing_trace 缺失

## TC-V3.0.2-03：minimalist 流量预警

**修复内容**：首次选择 minimalist 时提示互动率下降风险；用户显式指定时跳过预警。

**测试输入 A（自动路由到 minimalist）**：
```
用户名：摄影师小陈
输入：离开北京的那天，没有告别
```
- 用户名含"摄影" → 路由到 minimalist

**验证点 A**：
1. 首次路由到 minimalist 时，输出流量预警：
   > "该风格偏文学克制，可能导致互动率下降 30-50%。建议：保留核心情感但增加 1-2 个情绪钩子。回复'继续'使用白描，回复'加点情绪'自动补充"
2. 用户回复"继续" → 按白描模式生成
3. 用户回复"加点情绪" → 切换到 bff 并保留白描核心
4. 同一会话第二次选择 minimalist → 不重复预警

**测试输入 B（用户显式指定 minimalist）**：
```
帮我写一条情感笔记，用白描风格
```

**验证点 B**：
1. 用户显式指定 minimalist → 跳过预警（用户已知风险）
2. 直接进入白描模式生成

**验收标准**：
- ✅ PASS：自动路由时显示预警 + 显式指定时跳过预警
- ❌ FAIL：自动路由时不显示预警，或显式指定时仍显示预警

## TC-V3.0.2-04：信息完整性闸门（快速→教练模式）

**修复内容**：快速模式→教练模式切换时，校验最低有效信息量，避免"升级反而变差"。

**测试输入 A（信息不足）**：
```
用户第一次输入：帮我写一条好物推荐，珀莱雅双抗精华
（快速模式生成，仅产品名，其他用 📌[待补充] 占位）
用户第二次输入：深度优化
```

**验证点 A**：
1. 系统检查 Fact Sheet 信息量：好物推荐最低要求（产品名✅ + 核心效果❌ + 个人感受❌）= 1/3 < 2/3
2. 不直接进入教练模式评分
3. 先补充采集 1-2 个关键问题（如"用了多久？效果怎么样？"）
4. 补充采集不超过 2 轮
5. 用户说"不用了，直接评分" → 跳过采集，评分中标注"信息不足，评分仅供参考"

**测试输入 B（信息充足）**：
```
用户第一次输入：推荐珀莱雅双抗精华，用了3个月，皮肤真的变好了，价格不到200
（快速模式生成）
用户第二次输入：深度优化
```

**验证点 B**：
1. 系统检查 Fact Sheet 信息量：好物推荐（产品名✅ + 核心效果✅ + 个人感受✅）= 3/3 ≥ 2/3
2. 直接进入教练模式，不重复采集
3. 输出完整五维评分 + 3 个标题变体

**验收标准**：
- ✅ PASS：信息不足时补充采集 + 信息充足时直接进入
- ❌ FAIL：信息不足时直接评分（无警告），或信息充足时重复采集

## TC-V3.0.2-06：可折叠风险标签（合规提醒升级）

**修复内容**：黄线提醒升级为可折叠风险标签，提供"合规润色"和"保留原文"两个选项。

**测试输入**：
```
帮我写一条好物推荐，推荐一款减肥茶，喝了之后瘦了5斤，排毒效果特别好，气色也好了
```

**验证点**：
1. 正文中的黄线表述用 ⚠️ 标记（如"瘦了""排毒""气色好了"）
2. 输出末尾提供两个选项：
   ```
   ⚠️ 检测到 N 处风险表述：
   1. 回复"合规润色" → 一键替换为安全表述
   2. 回复"保留原文" → 我已知悉风险，自行承担
   ```
3. 用户回复"合规润色" → 自动替换：
   - "瘦了" → "感觉轻盈了"
   - "排毒" → "身体感觉舒适了"
   - "气色好了" → "朋友说我看起来精神了"
4. 用户回复"保留原文" → 不修改，日志记录 `compliance_choice: "user_accepted_risk"`

**合规润色替换表验证**：
| 风险表述 | 安全替换 |
|---------|---------|
| 瘦了/减了/轻了 | 感觉轻盈了/体感变好了 |
| 排毒/清肠 | 身体感觉舒适了 |
| 治便秘/通便 | 消化感觉顺畅了 |
| 气色好了/皮肤好了 | 朋友说我看起来精神了 |

**验收标准**：
- ✅ PASS：风险标记 + 双选项 + 合规润色替换正确
- ❌ FAIL：无风险标记，或无双选项，或替换不正确
