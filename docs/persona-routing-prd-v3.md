# 《人设路由系统 PRD V3.0》

> 版本：V3.0 | 作者：@pm | 日期：2026-06-30
> 基于 V2.3 全员评审 96 条真实反馈数据分析

---

## 1. 问题定义

**核心痛点**：当前系统是"5 种内容类型 x 1 种默认风格"，默认"闺蜜聊天风"对非美妆/穿搭用户严重不适配。

**数据支撑**：

| 指标 | 数值 | 含义 |
|------|------|------|
| 低分反馈中"人设适配失败"占比 | **62%** | 近 2/3 的不满意源于风格错配 |
| 语域力平均分 | **B 档下沿（6.8-7.6）** | 默认风格无法覆盖多元博主需求 |
| 当前风格覆盖 | 8 种预设，1 种默认 | 预设存在但未被自动路由激活 |

---

## 2. 设计目标

**从"5 类型 x 1 风格"到"5 类型 x N 人设"**。

| 维度 | 当前 | 目标 |
|------|------|------|
| 风格系统 | 8 种预设，1 种默认 | 8 种人设，智能路由 |
| 路由方式 | 用户名关键词（5 条） | 多信号融合（用户名+内容+Fact Sheet+显式选择） |
| 语域力目标 | B 档下沿 | A 档占比 30%+，均值 8.0+ |
| 低分归因 | 62% 人设不适配 | <20% |

## 4. 路由规则

**优先级**：用户显式指定 > Fact Sheet 风格偏好 > 用户名信号 > 内容类型默认

**内容类型默认人设**：
| 内容类型 | 默认人设 |
|---------|---------|
| 好物推荐 | `seeder` |
| 教程攻略 | `lecturer` |
| 情感共鸣 | `bff` |
| 探店打卡 | `aesthete` |
| 对比测评 | `expert` |

**兜底**：不再无脑兜底到"闺蜜聊天风"，按内容类型默认人设路由。

---

## 5. 实现路径

| 阶段 | 交付物 | 验收标准 |
|------|--------|---------|
| P0（第一周） | 人设清单 + 路由规则 + 去AI味人设化 + 称呼人设化 | 自动路由到内容类型默认人设 |
| P1（第二周） | 多信号融合 + Fact Sheet增强 + persona字段 | 自动路由准确率≥70% |
| P2（第三周起） | 路由看板 + 自动调优 + A/B测试 | 人设不匹配<20%，语域力A档≥30% |

## 7. 风险评估

| 风险 | 缓解 |
|------|------|
| 人设过多选择困难 | 自动路由透明，用户无需主动选择 |
| 自动识别误判 | 多信号融合 + 手动覆盖权 |
| 语域力评分碎片化 | 按人设分组（高互动/高专业/高审美）定义标准 |
